CSR (Certificate Signing Request)

Content: Public Key, domain name, Organization Name, Location (city, state, country)

Purpose: Used to create an SSL certificate

CRT (Certificate)

This is the SSL certificate issued by a Certificate Authority. It is installed on the server to establish secure communication

Private key

contains the private key corresponding to the public key in the CSR and SSL certificate

It is used by the server to securely establish communication with clients

During the SSL/TLS handshake, the private key works with the public key in the certificate to encrypt and decrypt data.

Steps

1 - Generate .key (Private Key):

This is the first step. The private key is generated and kept secret.

2 - Generate .csr (Certificate Signing Request):

The CSR is created using the private key. It contains information about your domain and organization.

3 - Receive .crt (Certificate):

After submitting the CSR to a Certificate Authority (CA), the CA issues a certificate (.crt), which is tied to the private key.

2. Convert your certificate files into a .pfx file

1 - copy the .key and the .crt files into your WSL or another Linux machine.

2 - Open your WSL terminal and go to the folder you copied the .crt and .key files

3 - run the following command

openssl pkcs12 -export -out vConWild.pfx \
-inkey vConWild-generated-private-key.txt \
-in vConWild-chained-crt.txt

explanation:

-out vConWild.pfx: The name of the output .pfx file.

-inkey: Specifies your private key file.

-in: Specifies your certificate chain file.

4 - Specify a password for your .pfx file

5 - Your .pfx file is created and is located in the same folder you stored the .crt and .key files
