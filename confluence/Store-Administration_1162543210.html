vConestoga : Store Administration

Created by Nick Aguilar, last modified on Aug 15, 2024

Overview

This will act as a guide for pulling new code and restarting the server should a service change. This will be an ongoing documentation effort as we encounter various circumstances and the software environment continues to evolve.

Pull Software Update

  1. Connect to the server

Enter the following code into the terminal:

ssh -i .ssh/assetstore_key.pem shopkeeper@assetstore.canadacentral.cloudapp.azure.com

or

ssh -i .ssh/assetstore_key.pem shopkeeper@assetstore.vconestoga.com

  1. Ensure your console is in the varlab-asset-store folder

  1. git pull

Git saves information from the clone command, like which host and private SSH key are required to access the repository

git pull

  1. Build the project

Navigate to the client folder

Build the server using the following commands:

npm i           #Only if you have new packages to install
npm run build

Be sure to npm install both the client and server

This will cause the project to be built in the dist folder under client

Restart Services

If changes are made to the server code you need to restart pm2 which runs services for the client and the server.

Run the following command to refresh the server code with any new changes

pm2 restart server

You should see the following output:

Check pm2â€™s status to ensure no errors occurred when you restarted the server restarted.

pm2 status

Sometimes the Serve requires restarting as well, therefore the client can be restarted one of two ways

pm2 restart client

This restarts only the client

or

pm2 restart all

This restarts all services (in this case the server and the client

If you use the all command, you will also restart the meta search related services as well as any CI/CD services.

Edit ecosystem.config.js

If you need to edit the ecosystem.config.js (which contains configuration information for PM2) or if restart is not having the intended effect, execute the following commands

pm2 kill
pm2 start ecosystem.config.js

Ensure that your terminal is in the same directory as the ecosystem.config.js (in this case the projects root directory) before executing the second command.

NGINX

If changes are made to the servers proxy NGINX needs to be restarted.

Edit NGINX Config

Enter the following command into your console to edit the configuration file

sudo vi /etc/nginx/nginx.conf

Refresh NGINX

Once a change is made to the configuration file the service needs to be stopped and restarted. Enter the following commands:

sudo systemctl stop nginx
sudo systemctl start nginx
