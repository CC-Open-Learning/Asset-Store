vConestoga : Asset Storage

Created by Nick Aguilar, last modified on Aug 13, 2024

Overview

Assets are currently stored on the Linux server file system. Based on the estimated workload for the website, it was determined that the file system would be able to scale to meet demand.

The Assets in the server reside in the following directory.

/home/shopkeeper/asset-storage

This directory is currently made up of two sub-directories: Assets and Preview-Images

  • The Assets directory contains all asset files (currently without any sub-directories or sorting)

  • Preview-Images exists to store locally (to the server) various preview images we may need

  • Project-Images exists to store locally (to the serve) various preview images for each project

  • Only the test image and files are not sanitized, all other files stored on the server should be sanitized. This will effect downloads which will appear with the original file name

Local File usage

Should you wish to use the files locally, you can copy the original zip file from the Linux server (located in the home (~) directory. More specific instructions on how to run a local environment are located in Local Environment

  • To download SSH into the server

    • Follow this guide if you do not have it setup via VS Code Remote Development With VS Code

  • Make sure you SSH into the server at the shopkeeper level not directly into the code base

    • Your should see the following:

  • Find the asset-storage-original.zip file on the left panel and right-click on the file

  • Select download and wait for the download to complete

  • Unzip the file and ensure the file is named “asset-storage”

  • This should give you access to all the files you need to setup the demo version of the store

  • Ensure that you edit you local servers .env file to point at where you stored the asset-storage folder on your local machine by changing the following property

ASSET_STORAGE_ROOT=<local file path>
