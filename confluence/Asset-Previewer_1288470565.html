 How does it work?

How does the Asset Previewer - (AP) function?

  1. When the preview icon is selected in the Asset Details component, the AP downloads the respective file which is the high majority of cases, will be .zip compressed files. creates a new 3D world space using React Three Fiber, .

  2. AP unzips .zip files and loads in sequential order:

    • Textures files (png, jpg, jpeg, svg, tiff, tga, webp).

      • The program loads all the supported texture files that were in the zip file regardless of the folder location, adds them to the list of assets available for preview and a textures only list.

    • Models files (glb, gltf, fbx, dae, obj (Still need MTL loading functionality)).

      • On loading of the model, the program will traverse through all meshes to get all materials and then will use the textures that were loaded prior in the textures loading to populate all the correct maps to the correct materials and mapping type.

        • Naming Conventions: The texture names need to match the material names they belong to.

          • Model - Asset_Name

          • Material - Asset_Name_Mat, Asset_Name_Alpha

          • Textures - Asset_Name_color, Asset_Name_normal, etc. after the last under score it reads the word to match the proper map in the material.

    • Production Files(blend, ma, mb, 3ds, max, psd).

      • This is currently disabled and not fully implemented, but the plan for this is similar to above. The production file will display a loaded model file that matches the proper naming conventions.

        • BLEND_FILE_NAME.blend

        • BLEND_FILE_NAME.glb <---- it will look for this to display a preview for the

      • Note: psd gets handled like a texture.

  1. All the loaded textures and textured models get added to an asset list that is valid for displaying in THREE.js. You can cycle through all the assets using the “green“ button. When viewing a texture type object, you can use the “yellow“ button to cycle the geometric shape that the texture has been applied for displaying.

  2. You can still preview single files but there is no support for any files that have other file dependencies. i.e. a model needing texture files.
