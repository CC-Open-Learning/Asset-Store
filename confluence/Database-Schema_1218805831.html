vConestoga : Database Schema

Created by Nick Aguilar, last modified by Islam Ahmed on Jan 17, 2025

Overview

This documentation will outline specific information regarding the schema of the database.

MongoDB was chosen as the database for this project in order to keep the database technologies consistent with the other vConestoga databases which also rely on MongoDB

MongoDB

The database for the server is a NoSQL MongoDB database with two collections shown in the image below

The Asset collection is the schema current schema for how assets of various types are stored in the database. The User collection shows how user information is stored.

Schemas

Asset Schema

Property Name

Data Type

Description

_id

ObjectId

Unique identifier for each asset generated by the MongoDB database upon insertion

fileName

Number

The full file name of the asset including its extension.
ex. Airtank_DClass.fbx

(this property is used to retrieve the file from storage)

fileSize

Number

The total size of the file in bytes

description

String

General description of the asset

format

String

The file extension for each file

name

String

Name of the asset (used for keyword search)

price

Number

The monetary value of the asset (if 0 assume FREE)

liscense

String

A field to enter any associated licensing information

origin

String

Where the asset came from, for our Assets VARLab

previews

String Array

An array of strings containing URLs to a given image
Ex. https://assetstore.vconestoga.com/api/image/<imagename>

tags

ObjectId []

array of strings containing tags to be used to search for a given asset. These tags should describe the asset and its various associations to a project

createdAt

Date

The time that the object was added to the database

updatedAt

Date

timestamp for the last time that the asset was updated

category

ObjectId []

Array containing references to category ID (see Asset-Category Schema for more information)

 

projects

ObjectId []

Array containing references to project ID (see Project Schema for more information)

model

Model

See model schema for more information about models, this is an optional field that can be contained in the asset schema if the asset is a model

texture

Texture

See texture schema for more information about textures, this is an optional field that can be contained in the asset schema if the asset is a texture

production

Production

See production schema for more information about production, this is an optional field that can be contained in the asset schema if the asset is a production

IMPORTANT FOR COMMUNICATING WITH THE API
For category, projects and tag fields, when these fields are requested from the API, they will include the name (as a string) of the property along with its ObjectID

User Schema

Property Name

Data Type

Description

username

String

The username for each user

hash

String

It is the result of combining the password and the salt after the encryption process. This is the value used to check if the password is correct and authenticate the user

salt

String

It is an unique and random data generated for each user. it Is used in the hash function to guarantee that the encryption pattern can not be discovered

googleId

String

The unique identifier provided by google oauth

picture

String

contains the URL of the user profile picture. If google oauth is used, it uses the Google profile picture, otherwise, it is set to ‘default’

refreshToken

String

refresh token
