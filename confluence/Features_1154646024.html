vConestoga : Features

Created by Anzhelika Kostyuk, last modified by abielaski on Jun 18, 2024

Prefered Side Panel Information for each asset

3D Model formats

FBX, GLB, DAE, OBJ

Image formats

PNG, JPEG, TIFF, TGA

Production file formats

blend, ma, mb, psd, sp

Geometry Information

Polygons: ex.1347

Vertices: ex.10376

Triangles: ex.3987

Edges: ex.2347

LODs

num. of LODs

(0 if we don’t have any LODs; if we have LODs present, we just put the total number; we can add geometry information for each LOD in the description field to keep our side panel information clean)

Animations

num. of animations

(In the description field: Action-based or Timeline-based + breakdown of animations, for example: idle, walk, run, crawl, fly, jump, etc)

Rigged geometries

Yes/No

(In the description field: FK type or IK type)

Textures

total count

(breakdown of textures in the description field)

Art style

PBR or PBR/MRO or Cel shade etc.

Asset Store Features

Sorting/searching

  • Sort/Search by Project (1 asset can belong to multiple projects)

  • Search by keywords/tags

Storage formats

  • Store and recall model formats (FBX, GLB, DAE, OBJ)

  • Store and recall image(texture) formats (PNG, JPEG, TIFF, TGA)

  • Store and recall production file formats (blend, ma, mb, psd, sp, and more)

Model Preview

A 3D model viewer or a few screenshots/renders of the piece from Blender/Unity. Likely this should be something we upload that is generated manually as auto generating them might not give a valuable preview (which is something we see in Unity a lot).

Additional information about the model

  • Geometry Information (tri counts, vertex counts, polys)

  • Listing of LODs associated with the model

  • List if Asset has been Rigged and/or Animated. Including information on if the Animations are recorded as strictly Timeline based or if they have been setup as Actions

  • Listing of included textures (_colour, _ao, _normal, etc)

  • Listing of applicable Material Styles (PBR/MRO, Cel Shade, etc)

Supporting Files storage

A section to store supporting files (shader graphs)

Versioning

A historical version of an asset in case we need to roll back.
