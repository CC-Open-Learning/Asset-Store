vConestoga : ❌ HTTP Error Cases

Created by Yujin Jeong, last modified on Aug 14, 2024

This documentation outlines the error codes used in the project

Error Status Code

Description

Source

400

the server could not understand the request because of invalid syntax.

  • file upload handler middleware( app.use ) in upload.js

    • Multer error

  • POST call to create new project in projects.js

    • Multer error

    • project name exists

  • /image

    • Multer error

  • /toggleUserRole

    • username does not exist in the request body

401

lack of valid authentication credentials for the requested resource

  • /refreshtoken

    • cookie is missing or improperly formatted

    • token cannot be verified

    • passport authentication failed

  • /login

    • cannot find user in the database

    • user enters wrong password

  • authenticateWithPassport()

    • authentication with the provided access token has failed

  • isContentTypeValid()

    • content-type is not “multipart/form-data”

403

the server understood the request but refused to process it

  • checkIsAdmin()

    • when user’s role is not admin

  • /allUserRoles

    • user is not using admin account

  • /toggleUserRole

    • user is not using admin account

    • admins wants to change their role to user

404

the server could not find the requested resource

  • delete asset router “/”

    • the asset id is not found from the request body

  • /toggleUserRole

    • username does not exist in the database

500

the server encountered an unexpected condition that prevented it from fulfilling the request

  • all /asset routes

  • all /project routes

  • all /search routes

  • file upload handler middleware(app.use) in upload.js

    • unexpected errors other than Multer errors during the upload process
