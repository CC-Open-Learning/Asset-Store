vConestoga : Azure Blob Storage

1 - How to create a blob storage

  1. Login to Azure

  2. Search for “Storage accounts”

  3. Click “Create”

  4. Select the subscription you want to use

  5. Select the resource group you want to add the blob storage to (Asset Store)

  6. Choose a unique name

  7. Select primary service to “Azure Blob Storage or Azure Data Lake Storage Gen 2”

  8. Change “Redundancy” to “Locally-redundant storage (LRS)”

  9. Click next

  11. Change the access tier to “cool.” Leave the other values as default.

  12. click Review + create

  14. Click Create

2 - Initial settings

  1. Go to the storage account you just created

  2. Settings >> Resources sharing (CORS)

  3. Set a new CORS rule to allow the client to access it:

    1. Allowed origins = http://localhost:5173 (Add any other domain names that you’re using for test and production servers here)

    2. Allowed methods = GET, DELETE, POST, PUT

    3. Allowed headers = *

    4. Exposed headers = *

    5. Max age = 3600

  4. click Save

3 - Create "folders" (containers)

  1. Go to Data storage >> Containers

  2. Click on ➕Container

  3. Name it “assets”

  4. click on Create

  5. Create two more containers: “preview-images” and “project-images”

4 - Create .env variables

  1. Create 4 new variables in the .env file located in the server folder:

    1. BLOBSTORAGE_URI

    2. AZURE_STORAGE_ACCOUNT_NAME

    3. AZURE_STORAGE_ACCOUNT_KEY

    4. AZURE_STORAGE_CONNECTION_STRING

  2. Get the values from the Azure portal

  3. Go to the storage account

  4. Go to “Security + networking” >> Access keys

  5. Copy Storage account name

  6. Use the values from key 1 and ignore key 2

  7. Click show and copy Storage account key

  8. Click show and copy Storage account connection string

  9. The BLOBSTORAGE_URI will use your account name in this format:

    1. BLOBSTORAGE_URI=https://varlabstorageaccount.blob.core.windows.net/
